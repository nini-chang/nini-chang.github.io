Setup Hugo Blog With Github Page

April 15, 2021 2-minute read
hugo • blog

Hugo + GitHub Page

  1. 安裝 Hugo
  2. 建立 Hugo 專案
  3. 新增 Hugo 主題
  4. 啟動 Hugo 專案
  5. 將 Hugo 專案部署至 GitHub Page

1. 安裝 Hugo

  • For Mac (HomeBrew): $ brew install hugo
  • 可參考官方文件: 安裝 HUGO

2. 建立 Hugo 專案

  • 產生 hugo repo: $ hugo new site mgHugoBlog
    • mgHugoBlog 可以替換成你要的名稱
    • 裡面會有 content、data、layouts、resources、static 等資料夾
  • 完成上述步驟後，就已經有基本的架構了

3. 新增 Hugo 主題

  • 可參考官網提供的 Hugo 主題
  • 進去專案、初始化、新增主題(這邊以 Hugo-coder 為例)
    • $ cd myHugoBlog
    • git init
    • git submodule add https://github.com/luizdepra/hugo-coder.git themes/hugo-coder
  • 專案裡面就會多一個資料夾是 themes 那包東西就是你剛剛新增進去的主題
    • 有些會附上設定的範例可以參考
    • 其中，我的 config.toml 是參考這個主題的進行配置，可參考: hugo-coder/exampleSite/config.toml

4. 啟動 Hugo 專案

  • 執行 $ hugo server 或是 $ hugo server -D就可以啟動專案囉

    • 開啟 https://localhost:1313/ 就會看到網站目前的樣子

  • 如果跑不起來，可能要看一下錯誤訊息、檢查 hugo 版本

    • 查看錯誤訊息: $ hugo --debug
    • 檢查版本: $ hugo version (延伸閱讀: Hugo 安裝指定版本)
    • 經驗分享: 我之前遇到一次是因為 config.toml 沒有設定好，導致檔案一直找不到

  • 補充: 如果要新增文章，可以直接下指令 $ hugo new posts/my-first-post.md (這個檔案會長在 content/posts 裡面)

5. 將 Hugo 專案部署至 GitHub Page

  • 打包程式碼: $ hugo
    • 會產生一個資料夾 public ，這就是部署至 GitHub page 時要用的
    • 到目前為止結構會長這樣
  • 到 GitHub 新增一個 repo
    • 以我的為例: GitHub 名稱是 nini-chang。那麼這個 repo 就要命名為 nini-chang.github.io 才能夠使用 https://nini-chang.github.io 這個網址
    • 建立的時候記得要勾選 create README.md，否則他不會幫你創建分支（印象中）、repo 要設為 public
    • 建立後要到 settings -> 左側欄的 pages -> source 中指定 branch: main & /(root)
    • 但後來發現其實設定的方法不只一種，可以參考這篇
  • 完成後，回到本地端的 hugo 專案
    • 先進去我們剛打包的資料夾: $ cd public
    • 初始化、設定 remote repo (就是剛剛在 GitHub 新增的 nini-chang.github.io)
      • $ git init
      • $ git remote add origin git@github.com:nini-chang/nini-chang.github.io.git
      • 檢查一下是否有設定成功: $ git remote -v (會出現 fetch 跟 push 的 origin)
    • 推上去之前要注意一下 branch 的名稱是否為 main
      • GitHub 從去年開始就將預設的主分支從 master 改為 main
      • 如果是 master，請在該分支上輸入指令: $ git branch -m main
      • 完成後就會看到 branch 被改成 main 囉
    • 接著就是將剛剛打包的 public 推上去
      • $ git add .
      • $ git commit .
      • $ git push -u origin main

基本上就已經完成部署囉！可以開啟你的 xxx.github.io 看看是否有畫面跟文章了！

補充

如果想要將步驟二中建立的 hugo 專案上傳至 GitHub 維護，也可以開一個新的 repo (可設為 private)
