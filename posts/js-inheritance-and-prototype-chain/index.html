JavaScript 類似繼承設計與原型鍊

July 14, 2021 · 3 min · Nini Chang

1994 年，Netscape 發布 Navigator browser version 9.0。第一個較成熟的網路瀏覽器，但此版本只能「瀏覽」。工程師 Brendan Eich 負責使瀏覽器可以與網頁互動的語言，他認為只要能夠完成一些簡單操作就夠了。

1994 年：Object-oriented programming 最興盛的時期，C++ 是當時最流行的語言，Java v1.0 即將推出。這使 Brendan Eich 受到影響，JavaScript 中所有的數據類型都是 object（這點與 Java 非常相似）。

難題一：是否要設計「繼承」機制？#

基於 JavaScript 中都是 object，Brendan Eich 認為必須要有一種機制將他們聯繫起來，因此選擇設計「繼承」機制。但不打算引入 class 的概念，一旦有 class 使 JavaScript 成為完整的語言，但 Brendan Eich 認為這樣過於正式，且增加學習的難度。

第一步：將 new 指令引入，並調用 constructor（未引入 class）#

考量到 C++ 和 Java 都是使用 new 指令，生成實例（instance）。

  • Ｃ++ 語法: ClassName *object = new ClassName (param) ;
  • Java 語法: Foo foo = new Foo () ;

但 JavaScript 中沒有 class，Brendan Eich 想到 C++ 和 Java 在使用 new 指令時都會調用 class 的函式建構式（function constructor）。於是在 JavaScript 中設計：new 指令後方的是函式建構式（function constructor），而非 class。

// constructor
function dog (name) {
  this.name = name;
}

var dogA = new dog ('charlie');
console.log (dogA.name); // 'charlie'

實例：new 出一個 dog 的 instance — dogA。

new 的限制：無法共享屬性與方法#

用函式建構式（ constructor）生成實例對象（instance object）時，無法共享屬性（property）與方法（method），每個實例對象都有自己的屬性和方法副本，似乎有些浪費資源空間。

// constructor
function dog (name) {
  this.name = name;
  this.species = 'Canine';
}

var dogA = new dog('Charlie');
var dogB = new dog('Brownie');

dogA.species = 'Feline';
console.log(dogB.species); // 'Canine'

實例：dogA 和 dogB 是兩個不同的 function。

第二步：將 Prototype 屬性引入#

Brendan Eich 考量到上述資源無法共享造成的浪費問題，決定為函式建構式（constructor）設置一個 __ proto__ 屬性。這個屬性包含一個對象（Object.prototype）— 所有實例對象需要共享的屬性和方法都在這個對象中，不需要共享的屬性和方法就留在函式建構式當中。

實例對象一旦建立，便會自動引用 Object.prototype 的屬性和方法。

/* 改寫上述的範例 */

function dog (name) {
  this.name = name;
}

dog.prototype = { species: 'Canine' };

var dogA = new dog('Charlie');
var dogB = new dog('Brownie');

console.log(dogA.species); // 'Canine'
console.log(dogB.species); // 'Canine'


/* 只要修改 object.prototype 就會同時影響到兩個實例 */

dog.prototype.species = 'Feline';

console.log(dogA.species); // 'Feline'
console.log(dogB.species); // 'Feline'

實例：new 出兩個 dog 的 instance：dogA、dogB，並在 dog.prototype 上加上欲共享的屬性（species: ‘Canine’）。亦可加上方法共享之。

也因為所有的實例對象共享同一個 Object.prototype，會讓人產生 Object.prototype 就是實例對象的原型，而實例對象似乎「繼承」了 Object.prototype 的錯覺。

記住：其實每一個函式（function）都有一個 prototype 屬性，通常是一個空的對象（an empty object）。

constructor 與 instance 的連接方式：proto#

以實例（dogA、dogB 與 dog 的連接方式）來說明：

function dog ( name, species ) {
  this.name = name;
  this.species = species;
}

dog.prototype.log = function ( ) {
  console.log(this.name + ', species: ' + this.age);
}

var dogA = new dog ( 'Charlie', 'Canine');

console.log(dogA.__proto__ === dog.prototype); // true

備註：其實比較好的方式是用 Object.getProtoypeOf ()

實例：dogA 的 proto 會指向 dog.prototype。當 JavaScript 在 dogA 中沒找到 log 方法時，便會透過 proto 找到 dog.prototype，查看 dog.prototype 中有沒有 log 這個方法。

假設 dog.prototype 中沒有 log 方法，JavaScript 會繼續依照此規則去查找 dog.prototype.proto 中有沒有 log 方法，不斷地查找。直至找到某個東西的 proto 是 null 才停止，表示這就是原型最頂端了。

原型鏈（Prototype chain）#

透過 proto 不斷串起來的就叫做原型鏈，藉此可達成「類似繼承」的功能，呼叫 parent method / property。

function dog (name, species) {
  this.name = name;
  this.species = species;
}

dog.prototype.log = function ( ) {
  console.log(this.name + ' , species' + this.species);
}

var dogA = new dog ('Charlie', 'Canine');

console.log(dogA.__proto__ === dog.prototype); // true
console.log(dog.prototype.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__); // null

實例：dogA.proto 指向 dog.prototype 、dog.prototype.proto 指向 Object.prototype、Object.prototype.proto 指向 null，表示這為原型鏈的最頂端。

/* Prototype chain */

dogA.__proto__ == dog.prototype;
dogA.__proto__.__proto__ == dog.prototype.__proto__ == Object.prototype;

實例： dogA 對象繼承自 dog 的原型對象（dog.prototype），而 dog 的原型對象又繼承自 object 的原型對象，這樣就構成了原型鍊（prototype chain）。

/* 延續上述實例 */

console.log(dog.__proto__ === Function.prototype); // true
console.log(Function.prototype.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__); // null

實例：dog 其實就是一個 Function 的 instance。
